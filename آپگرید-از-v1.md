# آپگرید از v1 به v2 — قدم به قدم

این راهنما برای وقتیه که قبلاً نسخه v1 (با `checker.py` و `notify_if_before` در YAML) رو راه انداختی و حالا می‌خوای v2 (کنترل تلگرامی) رو بذاری.

زمان تقریبی: **۱۰-۱۵ دقیقه**

---

## ۱. یه backup سریع (اختیاری)

اگه می‌خوای امن باشی، در GitHub:
- Settings → General → پایین صفحه → **Archive this repository** → **No**
- یا فقط یه fork بگیر برای backup

ولی واقعاً نیاز نیست، چون commit history همه‌چی رو نگه می‌داره.

---

## ۲. فایل‌های جدید رو در ریپو جایگزین کن

**نکته مهم**: نسخه v2 ۸ تا فایل جدید/تغییرکرده داره. ساده‌ترین راه:

### روش الف — github.dev (راحت‌تر)

۱. در URL ریپوت `github.com/...` رو به `github.dev/...` تغییر بده (یا حرف `.` بزن)
۲. ادیتور باز می‌شه. در سمت چپ پنل Explorer، **پوشه `afspraak-watcher-v2` رو از Desktop به ادیتور drag کن**
۳. اگه پرسید "Replace existing files?" → **Replace**
۴. پنل Source Control (آیکون شاخه) → پیام: `upgrade to v2` → Commit & Push

### روش ب — حذف و آپلود جدید

اگه github.dev مشکل داشت:

۱. در ریپو، فایل‌های زیر رو **پاک کن** (روی هرکدوم کلیک → سطل آشغال):
   - `checker.py` (نسخه قدیمی)
   - `state/ridderkerk.json` (schema قدیمی)

۲. فایل‌های زیر رو با محتوای نسخه v2 **جایگزین کن** (روی فایل → مداد → پیست → Commit):
   - `sites/ridderkerk.yaml`
   - `.github/workflows/check.yml`
   - `README.md`
   - `requirements.txt`

۳. فایل‌های زیر **جدیداً** بساز (Add file → Create new file):
   - `checker.py` (نسخه v2)
   - `main.py`
   - `bot.py`
   - `state.py`
   - `sites.py`
   - `telegram_io.py`
   - `ADDING_NEW_SITE.md`

## ۴. اجرای دستی برای تست

۱. تب **Actions** → **Check appointments** → **Run workflow** (سبز)
۲. صبر کن ~۱ دقیقه. باید سبز بشه.
۳. روی ران کلیک کن → **check** → **Run main** → لاگ رو ببین.

باید چنین چیزی ببینی:
```
PHASE 1: process Telegram updates
Telegram updates pending: 0 (offset=0)
PHASE 2: check active sites
No active sites... (یا چیزی شبیه این)
```

اگر لاگ گفت "skipped (paused)" برای ridderkerk، نرماله — چون state جدید پیش‌فرض paused هست.

---

## ۵. اولین /watch از تلگرام

۱. تلگرام رو باز کن، چت بات
۲. بفرست: `/help` — باید راهنمای کامل بیاد
۳. بفرست: `/list` — باید ridderkerk با علامت ⚪ (paused) بیاد
۴. بفرست: `/watch ridderkerk` — بات تأیید می‌کنه که شروع کرد
۵. صبر کن ~۱۰ دقیقه برای ران بعدی cron (یا دستی Run workflow بزن از Actions)
۶. باید پیغام "🎯 اولین تاریخ آزاد: ..." بگیری (baseline)

از این به بعد فقط وقتی تاریخی **زودتر** از این پیدا شه، خبر می‌گیری.

---

## ۶. تنظیم deadline (اختیاری)

اگه می‌خوای فقط برای تاریخ‌های زودتر از مثلاً ۹ یونی notif بگیری:

```
/deadline ridderkerk 2026-06-09
```

از این به بعد:
- اگه تاریخ ۱۵ یونی پیدا شه → notif نمیاد (≥ deadline)
- اگه تاریخ ۵ یونی پیدا شه → notif میاد ✅

برای پاک‌کردن: `/deadline ridderkerk clear`

---

## ۷. وقتی قرارت رو گرفتی

دو راه:

**روش الف**: زیر آخرین notif دکمه‌ی **✅ گرفتم** رو بزن. بات می‌پرسه مطمئنی؟ تأیید کن.

**روش ب**: `/booked ridderkerk` بفرست. بات سؤال تأیید می‌فرسته.

از این به بعد چک خودکار خاموش می‌شه. اگه دوباره خواستی فعال کنی: `/resume ridderkerk` یا `/watch ridderkerk`.

## مشکلات احتمالی

### بات هیچ جوابی نمی‌ده
- مطمئن شو `TELEGRAM_CHAT_ID` در Secrets ست شده (مقدارش `108046309`)
- بفرست `/start` — اگه جواب نمیاد، یه ران دستی از Actions بزن (هر دستور تا ران بعدی cron پردازش نمی‌شه)

### "command ناشناخته" برای دستور درست
- چک کن نسخه `bot.py` در ریپو با محتوای v2 یکیه

### نوتیف baseline نمیاد بعد از /watch
- مطمئن شو در ریپو `state/ridderkerk.json` رو پاک کردی (schema قدیمی بود)
- یا یه ران دستی بزن و لاگ رو ببین

### تاخیر زیاد بین دستور و پاسخ
- نرماله. cron هر ۱۰ دقیقه ست. اگه می‌خوای فوری ببینی، Run workflow دستی بزن.
