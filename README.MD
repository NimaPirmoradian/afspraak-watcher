# راهنمای قدم‌به‌قدم (بدون نیاز به دانش Git یا برنامه‌نویسی)

این راهنما با فرض اینکه **هیچ‌چیز از Git/GitHub نمی‌دونی** نوشته شده. همه‌چی از طریق مرورگر و وب‌سایت GitHub انجام میشه — هیچ ترمینال یا کامند لاینی لازم نیست.

زمان تقریبی: **۱۵-۲۰ دقیقه**

---

## ✅ پیش‌نیاز

- یک حساب جیمیل یا ایمیل دیگه
- این پوشه (`afspraak-watcher`) با همه فایل‌هاش

---

## مرحله ۱ — ساخت حساب GitHub (اگر نداری)

1. برو [github.com/signup](https://github.com/signup)
2. ایمیل و رمز عبور بذار. یک username انتخاب کن (مثلاً `nima-piremorad`). این public میشه پس چیز خاصی نباشه.
3. ایمیل تأیید رو از inbox‌ت باز کن و کلیک کن.
4. وقتی پرسید "What's your role?" و "What do you want to do with GitHub?" هرچی خواستی انتخاب کن (مهم نیست).
5. پلن **Free** رو انتخاب کن.

---

## مرحله ۲ — ساخت ریپو جدید

1. لاگین کن، در بالا سمت راست روی **+** کلیک کن → **New repository**
2. در فرم:
   - **Repository name**: `afspraak-watcher`
   - **Description**: (هرچی، یا خالی)
   - **Public** را انتخاب کن (پیشنهاد) یا **Private** (با محدودیت ۲۰ دقیقه‌ای cron)
   - **مهم**: گزینه‌های زیر را **انتخاب نکن**:
     - ❌ Add a README file
     - ❌ Add .gitignore
     - ❌ Choose a license
   - دلیل: ما خودمون این فایل‌ها رو داریم و نمی‌خوایم تداخل کنه.
3. دکمه سبز **Create repository** را بزن.

حالا یک صفحه می‌بینی که میگه "Quick setup". اون رو نادیده بگیر و قدم بعدی رو دنبال کن.

---

## مرحله ۳ — آپلود فایل‌ها از طریق مرورگر

GitHub اجازه میده فایل‌ها رو drag-and-drop کنی، اما **پوشه‌ها رو نمیشه از طریق UI آپلود کرد**. این کار محدودیته. ما با یک ترفند ساده حلش می‌کنیم:

### روش الف: ساده‌ترین راه — استفاده از `github.dev` (ادیتور آنلاین)

1. در صفحه ریپوی خالیت، در URL بالا، حرف **`.`** (نقطه) رو روی کیبوردت بزن.
   - یعنی URL از `github.com/USER/afspraak-watcher` تبدیل میشه به `github.dev/USER/afspraak-watcher`
2. یک ادیتور شبیه VS Code باز میشه در مرورگر.
3. در نوار کناری چپ، روی **Explorer** (آیکون فایل) کلیک کن.
4. **پوشه `afspraak-watcher` رو از سیستمت با ماوس کشیده و در ناحیه فایل‌های ادیتور رها کن**.
   - همه فایل‌ها از جمله `.github/workflows/check.yml` (که در پوشه مخفی هست) آپلود میشن.
5. در نوار کناری چپ، روی آیکون **Source Control** (شبیه فلش انشعابی) کلیک کن.
6. در کادر بالای لیست فایل‌ها بنویس: `initial commit`
7. دکمه آبی **Commit & Push** را بزن. (یا گزینه `Commit` بزن بعد `Sync Changes`)
8. اگر پرسید "branch is published" یا چنین چیزی، تأیید کن.

### روش ب: اگر روش الف کار نکرد — آپلود از UI خود GitHub

این روش هم ممکنه ولی چون پوشه `.github/workflows/` رو باید دستی بسازی، کمی سخت‌تره:

1. در صفحه ریپو، دکمه **Add file → Upload files**
2. همه فایل‌های **سطح بالا** (`checker.py`, `requirements.txt`, `README.md`, `.gitignore`, `راهنمای-مبتدی.md`) رو drag-and-drop کن
3. پایین صفحه دکمه سبز **Commit changes**
4. حالا برای پوشه `sites/`: دکمه **Add file → Create new file**
   - در کادر اسم فایل بنویس: `sites/ridderkerk.yaml`
   - محتوای فایل رو (از فایل لوکالت) کپی-پیست کن داخل کادر بزرگ
   - Commit changes
5. همین کار رو برای `state/.gitkeep` و `.github/workflows/check.yml` تکرار کن
   - برای `.github/workflows/check.yml`: اسم رو دقیقاً همینطور تایپ کن، GitHub خودش پوشه‌ها رو می‌سازه

### تأیید: همه فایل‌ها آپلود شدن؟

بعد از آپلود، باید این ساختار رو در صفحه ریپو ببینی:

```
afspraak-watcher/
├── .github/
│   └── workflows/
│       └── check.yml
├── sites/
│   └── ridderkerk.yaml
├── state/
│   └── .gitkeep
├── .gitignore
├── README.md
├── checker.py
├── requirements.txt
└── راهنمای-مبتدی.md
```

اگر `.github/workflows/check.yml` رو نمی‌بینی، Actions کار نمیکنه. حتماً چک کن.

## مرحله ۵ — فعال‌کردن GitHub Actions

1. در ریپو، تب **Actions** (بالای صفحه)
2. ممکنه پیغامی ببینی: "Workflows aren't being run on this forked repository" یا "Workflows from this repository require approval"
   - دکمه **I understand my workflows, go ahead and enable them** رو بزن
3. در لیست workflow ها باید **Check appointments** رو ببینی.

## مرحله ۶ — تست دستی اولین اجرا

1. در تب **Actions**، سمت چپ روی **Check appointments** کلیک کن
2. در سمت راست، روی **Run workflow** کلیک کن
3. یک منو باز میشه — مستقیم روی دکمه سبز **Run workflow** کلیک کن (نیازی نیست فیلد input رو پر کنی)
4. صفحه رو بعد از ۵ ثانیه refresh کن. باید یک ران زرد یا چرخان (در حال اجرا) ببینی.
5. صبر کن ~۱-۲ دقیقه. اگر سبز شد ✅ یعنی موفق.
6. روی اون ران کلیک کن → روی **check** (job name) → روی **Run checker** → لاگ‌ها رو ببین.
   - اگر تاریخی پیدا شد، آخر لاگ می‌بینی: `Earliest available: 2026-06-XX`
   - اگر اون تاریخ زودتر از ۹ یونی بود، باید **تلگرام بگیری**.
   - اگر دیرتر بود، لاگ می‌نویسه: `No earlier slot found`.

## مرحله ۷ — تأیید اجرای خودکار

بعد از این از ۱۰ دقیقه دیگه به بعد، خودبه‌خود اجرا میشه. می‌تونی هر زمان به تب **Actions** بری و ببینی لیست رانش پر شده.

**نکته**: GitHub Actions cron همیشه دقیق نیست. ممکنه ران ۵-۲۰ دقیقه دیرتر اجرا بشه در ساعت‌های شلوغ. این محدودیت پلتفرمه.

## مشکلات رایج

### ❌ workflow اجرا نمیشه

- مطمئن شو فایل دقیقاً در مسیر `.github/workflows/check.yml` هست.
- در ریپو، تب **Actions** → باید **Check appointments** رو ببینی. اگر نه، فایل اشتباه آپلود شده.

### ❌ "Bad credentials" یا "401 Unauthorized" در لاگ تلگرام

- توکن یا chat_id اشتباهه. به Settings → Secrets برگرد و دوباره چک کن.
- می‌تونی Secret رو پاک کنی و دوباره بسازی (مقدار قبلی قابل دیدن نیست).

### ❌ "No date found on page" چندبار پشت سر هم

- ممکنه سایت موقتاً خراب باشه — معمولاً خودش درست میشه.
- اگر ۱۰ بار پشت سر هم fail شد، خودش بهت تلگرام میده.

### ❌ تلگرام نمی‌گیرم با اینکه ران سبزه

- یعنی تاریخ پیداشده **زودتر از `notify_if_before`** نیست. لاگ رو بخون — اگه نوشت `not earlier than deadline`، مشکلی نیست.
- اگر مطمئنی باید نوتیف بیاد، در فایل `sites/ridderkerk.yaml` چک کن `notify_if_before: "2026-06-09"` درست تنظیم شده باشه.

### ❌ خطای "Workflow file not parsing" در Actions

- محتمل‌ترین دلیل: indentation فایل YAML (تو، فاصله‌ها) به‌هم خورده. فایل رو از پوشه‌ی اصلی دوباره آپلود کن.

## تغییر تنظیمات بعداً

### تغییر تاریخ نوتیف

اگر وقتت رو زودتر گرفتی و میخوای حالا فقط برای زودتر از مثلاً ۲۵ مه نوتیف بگیری:

1. در ریپو، روی `sites/ridderkerk.yaml` کلیک کن
2. آیکون **مداد** (✏️ Edit this file) سمت راست بالا
3. خط `notify_if_before: "2026-06-09"` رو پیدا کن، تاریخ رو عوض کن
4. پایین صفحه: **Commit changes**

### حذف state (شروع از صفر)

اگر می‌خوای نوتیف‌های قبلی فراموش بشه و دوباره برای همه تاریخ‌های مناسب نوتیف بگیری:

1. وارد پوشه `state/` شو
2. فایل `ridderkerk.json` رو پاک کن (اگر وجود داره): روی فایل کلیک → آیکون سطل آشغال
3. Commit

### غیرفعال‌کردن موقت

اگر تا چند روز نمیخوای چک کنه:

1. **Settings → Actions → General**
2. در بخش "Actions permissions" → **Disable Actions**

برای فعال‌کردن دوباره: همین مسیر → **Allow all actions**

## اضافه‌کردن سایت جدید (در آینده)

این بخش وقتی به دردت میخوره که خواستی سایت دیگه‌ای (مثلاً شهرداری دیگه برای پاسپورت) رو هم چک کنی.

1. فایل `sites/ridderkerk.yaml` رو در ریپو باز کن، آیکون **Raw** → ctrl+A → ctrl+C (همه‌چیش رو کپی کن)
2. **Add file → Create new file**
3. اسم: `sites/rotterdam.yaml` (یا هر اسمی) — حتماً پسوند `.yaml`
4. محتوا رو پیست کن و این چیزا رو تغییر بده:
   - `name`: اسم نمایشی
   - `url`: URL سایت جدید
   - `steps`: ممکنه دکمه‌های سایت نام دیگه‌ای داشته باشن — مثلاً `'Volgende'` به جای `'Ga naar stap 2'`
   - `notify_if_before`: تاریخ موردنظر
5. Commit. تموم. ران بعدی این سایت رو هم چک میکنه.

اگر گیر کردی در این مرحله، یه چت جدید با من باز کن و فایل YAML + URL سایت رو بفرست — کمکت می‌کنم.
